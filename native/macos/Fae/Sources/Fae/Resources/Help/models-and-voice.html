Model & Voice Reference

This page explains Fae's voice/model behavior. Change settings in Settings > Voice.

Runtime Stack

STT: Parakeet

TTS: Kokoro-82M

LLM: Local embedded runtime by default (unless you configure API providers).

Voice Model Preset

The Voice Model setting controls which local Qwen voice model profile is used.

Auto: picks a model based on available RAM.

Qwen3-4B: strongest instruction quality, slightly slower startup.

Qwen3-1.7B: balanced quality/speed.

Qwen3-0.6B: fastest latency, lower quality.

Voice Identity

Enable Voice Identity: turns speaker matching on/off.

Assist mode: preferred mode for most users. Uses speaker matching but still allows direct-address fallback.

Enforce mode: stricter mode for shared spaces; only enrolled speaker is accepted for gated voice interaction.

Approval Hardening

Require Voice Match for Approvals means spoken "yes/no" approvals must match the enrolled voice profile.

If voice identity is disabled or no voice profile is enrolled yet, approval flow falls back to standard behavior.

Config Keys

These settings map to config keys in ~/.config/fae/config.toml:

voice_identity.enabled

voice_identity.mode

voice_identity.approval_requires_match

llm.voice_model_preset
