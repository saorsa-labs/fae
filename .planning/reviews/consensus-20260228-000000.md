## MUST FIX — 3+ Reviewer Votes

### 1. Unsafe Array Access: `NSScreen.screens[0]` in WindowStateController
**Files**: `WindowStateController.swift:78, 142, 188, 219, 281`
**Voters**: Error Handling, Kimi K2, Code Simplifier (3 votes)
**Severity**: CRITICAL — crash in headless/VM environments
```swift
// BAD
let screen = NSScreen.screens.first ?? NSScreen.main ?? NSScreen.screens[0]
// GOOD
guard let screen = NSScreen.screens.first ?? NSScreen.main else { return }
```

### 2. Force Unwrap: `.first!` in `FaeCore.swift`
**Files**: `FaeCore.swift:407, 417` (createMemoryStore, createSchedulerPersistenceStore)
**Voters**: Kimi K2, Code Simplifier, Error Handling (3 votes)
**Severity**: CRITICAL — crash if applicationSupportDirectory unavailable
```swift
// BAD
let appSupport = FileManager.default.urls(for: .applicationSupportDirectory, in: .userDomainMask).first!
// GOOD
guard let appSupport = FileManager.default.urls(for: .applicationSupportDirectory, in: .userDomainMask).first else { throw ... }
```
**Bonus**: Extract shared `faeDirectoryURL` helper to eliminate 3 duplicate lookups.

### 3. NotificationCenter Observer Leak in `FaeCore`
**File**: `FaeCore.swift` — `observeSchedulerUpdates()` observer not removed on deinit
**Voters**: GLM-4.7, MiniMax, Kimi K2 (3 votes)
**Severity**: HIGH — memory leak if observeSchedulerUpdates() called multiple times
```swift
// GOOD: store token and remove on deinit
private var schedulerObserverToken: NSObjectProtocol?
// In observeSchedulerUpdates():
schedulerObserverToken = NotificationCenter.default.addObserver(...)
// In deinit:
if let token = schedulerObserverToken { NotificationCenter.default.removeObserver(token) }
```
