Generated by `scripts/prepare_training_data.py` from `claude-post-train-data.md` (DPO pairs) and `codex-post-train-data.md` (SFT seed examples).

Run that script to regenerate:

```bash
python3 scripts/prepare_training_data.py
```

The source of truth is the markdown files. The JSONL files are derived outputs and are not committed to git.

## Files produced

- `dpo.jsonl` — 500 DPO preference pairs (prompt / chosen / rejected). Each line is a JSON object.
- `sft.jsonl` — SFT chat examples in messages format. Each line is a JSON object with a `messages` array.
- `dpo_train.jsonl` / `dpo_val.jsonl` — 90/10 split of DPO pairs (with `--split` flag).
- `sft_train.jsonl` / `sft_val.jsonl` — 90/10 split of SFT examples (with `--split` flag).
